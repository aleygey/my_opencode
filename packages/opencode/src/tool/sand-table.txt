Run a sand table exercise where a planner agent and an evaluator agent iterate on a workflow plan.

The planner generates a detailed plan based on the topic and context. The evaluator reviews it critically and either approves or requests revisions. This loop continues until the plan is approved or max rounds are reached.

Both agents use different models when available to maximize diversity of perspective.

Parameters:
- topic: The planning question or goal. ALWAYS format as structured Markdown, not a single run-on paragraph. Use a short title, then bulleted sections for goal / requirements / constraints / deliverables. Example:
    ## Goal
    Design a scheduled task system.

    ## Core requirements
    - Users can create/manage tasks (cron or interval mode)
    - Trigger execution at the scheduled time
    - Report results via desktop notification

    ## Constraints
    - Must match existing TypeScript/Bun monorepo style
- context: Additional context from the current session (optional, also Markdown).
- max_rounds: Maximum iteration rounds, 1-5, default 3.
- models: Override model assignment for planner/evaluator roles (optional; auto-detected from available providers if omitted).

The UI renders `topic` as Markdown in the plan view — structured input means the user (orchestrator) can scan the discussion topic at a glance.

After completion, you can use `msg_read` to review the full discussion history and `msg_write` to inject additional user context into an active discussion.
